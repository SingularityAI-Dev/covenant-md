# Trademark and Name Use

"COVENANT.md" is a name used by SingularityAI-Dev to identify this specification
and its reference framework. The specification text is openly licensed under
CC-BY-4.0 and the reference code under MIT, but the name itself is not part of
either grant.

## What you may do

- Refer to the specification by name in documentation, articles, and talks.
- State that your tool reads, writes, or works with COVENANT.md files.
- Fork and adapt the specification under CC-BY-4.0, provided your derivative is
  clearly named so as not to imply it is the canonical COVENANT.md.

## What requires care

You may describe a tool as **"COVENANT.md compliant"** only when all of the
following hold:

1. It implements the normative requirements of a published COVENANT.md version
   (every MUST in the version it claims), and names that version.
2. It validates real COVENANT.md files and reports conformance honestly, with
   no false positives on documents that violate a MUST.
3. It does not extend the file format with incompatible fields under the
   COVENANT.md name, and does not present non-conformant behaviour as conformant.

A tool that meets these conditions may state, for example, "COVENANT.md v1.0
compliant". A tool that partially implements the spec should say so plainly
(for example, "reads COVENANT.md") rather than claiming compliance.

## What you may not do

- Use "COVENANT.md" as the name of a competing or forked specification in a way
  that implies it is the official one.
- Use the name to endorse or promote a product without SingularityAI-Dev's
  permission.

Questions about name use: open an issue at
https://github.com/SingularityAI-Dev/covenant-md.
